# 🌍 WorldHub — نموذج أولي (Prototype)

هذا الموقع بُني من الصفر اعتماداً على وثائق التخطيط الثلاث (الفكرة، الهيكل، مخطط الملفات) والصورة المرجعية للتصميم، لأنه لم يكن هناك أي أكواد HTML/CSS/JS فعلية مرفوعة لتعديلها.

## المحتوى
- `index.html` — الصفحة الرئيسية (Feed)
- `profile.html` — الملف الشخصي
- `messages.html` — الرسائل (محادثات + دردشة)
- `notifications.html` — الإشعارات
- `world.html` — صفحة عالم واحد (Programmation)
- `dashboard.html` — لوحة التحكم والإحصائيات
- `create-post.html` — إنشاء منشور
- `jobs.html` — الوظائف
- `events.html` — الفعاليات
- `companies.html` — الشركات
- `marketplace.html` — سوق الخدمات (Marketplace)
- `assets/css/main.css` — نظام التصميم الكامل (الألوان، الخطوط، المكونات) مطابق للصورة المرجعية
- `assets/js/app.js` — القائمة الجانبية، الشريط العلوي، البيانات التجريبية (Mock Data)، والتفاعلات (إعجاب، متابعة، القائمة على الجوال)

## طريقة التشغيل
افتح `index.html` مباشرة في المتصفح، أو شغّل خادم بسيط:
```
python3 -m http.server 8080
```
ثم افتح `http://localhost:8080`

## ربط الموقع بقاعدة بيانات حقيقية (Supabase / Firebase)
كل البيانات حالياً وهمية (`MOCK` في `app.js`) لتوضيح الشكل والتفاعل فقط.
في أسفل `app.js` يوجد كائن `api` يحتوي على دوال جاهزة (`getFeed`, `getWorlds`, `createPost`, `toggleFollow`...) بنفس الأسماء التي يستخدمها الواجهة — كل ما عليك فعله هو استبدال محتوى كل دالة باستدعاء Supabase/Firebase الحقيقي دون تعديل باقي الصفحات.

## ملاحظات
- التصميم متجاوب (Desktop / Tablet / Mobile) مع قائمة جانبية قابلة للطي على الشاشات الصغيرة.
- الأيقونات مبنية بـ Emoji لتفادي الاعتماد على مكتبات خارجية — يمكن استبدالها لاحقاً بمكتبة أيقونات (Lucide/Font Awesome) بسهولة.
- الخط المستخدم: **Inter** (يُحمَّل من Google Fonts).
- هذا نموذج أولي (Frontend فقط) بدون خادم أو مصادقة فعلية أو قاعدة بيانات — الأولوية كانت لمطابقة الشكل والتجربة البصرية للصورة المرجعية مع بنية جاهزة للربط الخلفي لاحقاً.
